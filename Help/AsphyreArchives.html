Description

Asphyre Archive format with its functions and utilities that can be used for storing images, textures, and application's data. The information is compressed to save disk space and optionally encrypted to ensure confidentiality and protect the contents from unauthorized usage.

Overview

Classes, Interfaces, Objects and Records

Name Description
Class TAsphyreArchive Asphyre archive implementation class that provides methods for reading, writing, deleting and renaming records in compressed (and possibly encrypted) archive; Zlib is used for data compression, CRC32 is used for data integrity checks and Asphyre's native 128-bit XTEA cipher is used for data encryption.

Description

Types

TArchiveOpenMode = (...);

Handling mode for Asphyre archives that indicates possible usage scenarios and limitations.

Values
  • aomUpdate: The archive will be used both for reading and writing.

  • aomReadOnly: The archive will be used for reading data. No modifications are allowed.

  • aomOverwrite: The archive will be overwritten with blank new file; once this is accomplished, the mode will be changed to aomUpdate.

TArchiveRecordType = (...);

Explicit type of the archive's record, which determines how the data should be interpreted.

Values
  • artFile: The record is a file or binary data.

  • artImage: The record is a native Asphyre image that has been prepared and pre-formatted for loading directly to hardware's memory.

  • artFont: The record is a binary Asphyre font that has been optimized for performance and has its accompanying image data prepared to be loaded to hardware's memory.

TArchiveTypeAccess = (...);

File access type when handled by Asphyre, indicating how the archive's file name should be interpreted.

Values
  • ataAnyFile: The file name is a typical name as used by the underlying OS, which should include its full path.

  • ataResource: The file name is the name of resource that is integrated directly to the application's executable. Only the actual name will be used for resource name, any existing paths from the name will be stripped.

  • ataPackaged: The file name which is distributed in the same folder as the application's executable or within the same bundle on Mac OS. The file name will be treated as regular file name with any existing paths stripped and the path to the executable added.

Variables

ArchiveHInstance: SizeUInt;

The instance handle of the application's thread. This parameter must be specified when ArchiveTypeAccess is set to ataResource for reading data from resources. Typically, it should be set to application's hInstance.

ArchiveTypeAccess: TArchiveTypeAccess;

Global access type for any instance of TAsphyreArchive, controlling how the archive's file name is interpreted.
