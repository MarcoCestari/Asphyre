Description

Variables

EventAsphyreCreate: TEventProvider;

Asphyre creation event, where all Asphyre components should be created.

EventAsphyreDestroy: TEventProvider;

Asphyre release event, where all Asphyre components should be released.

EventAudioCreate: TEventProvider;

Sound device creation event, which occurs right after the audio device has been initialized. In this event it is possible to load some essential sound samples and music.

EventAudioDestroy: TEventProvider;

Sound device finalization event, which occurs right before the audio device is to be finalized to release all other device-dependant sound resources such as samples and music.

EventBeginScene: TEventProvider;

Start of rendering scene event, which occurs when the scene is being prepared to be rendered. In this event the necessary device states can be updated.

EventButtonClick: TEventProvider;

Button click event, which occurs when one of GUI buttons have been clicked. Typically, EventControlName and EventControlForm are set to define what control on which form has been clicked.

EventControlForm: string;

This variable usually contains the name of the form, which contained the control that sent the latest event notification, such as button click.

EventControlName: string;

This variable usually contains the name of the control that has sent the latest event notification, such as button click.

EventDeviceCreate: TEventProvider;

Device creation event, which occurs right after the device has been initialized. In this event it is possible to load some essential artwork and fonts.

EventDeviceDestroy: TEventProvider;

Device finalization event, which occurs right before the device is to be finalized to release all other device-dependant resources such as images and fonts.

EventDeviceInit: TEventProvider;

Device initialization event, where device and swap chain configuration should be specified. This event occurs right before Asphyre device is about to be initialized.

EventDeviceLost: TEventProvider;

Device lost event, which occurs either before the device is to be finalized or when the device has been lost. In this event all volatile device-dependant resources should be released.

EventDeviceReset: TEventProvider;

Device reset event, which occurs either after device has been initialized or recovered from lost scenario. In this event all volatile device-dependant resources should be created.

EventEndScene: TEventProvider;

End of rendering scene event, which occurs when the scene has finished rendering and is about to be presented on the screen (or render target). In this event it is necessary to finish all cached rendering processes.

EventTimerReset: TEventProvider;

Timer reset event, which occurs when a time-consuming operation has taken place so it is necessary to reset the timer to prevent stalling.
